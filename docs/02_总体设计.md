# 02 总体设计

## 2.1 系统架构

```

│                    用户浏览器                          │

│  │         HTML + CSS + JavaScript                 │ │

└─────────────────────────┼────────────────────────────┘
                          │ HTTP (JSON)
                          ▼
┌─────────────────────────────────────────────────────┐
│                  Flask Web 后端                       │
│  ┌──────────────┐  ┌──────────────┐  ┌────────────┐ │
│  │  app.py      │  │ ai_service.py│  │ file_      │ │
│  │  (路由控制)  │──│  (AI 调用)   │  │ storage.py │ │
│  │              │  │              │  │ (文件存储)  │ │
│  └──────────────┘  └──────┬───────┘  └────────────┘ │

│                    │  API         │                   │
│                    └──────────────┘                   │
└─────────────────────────────────────────────────────┘
```

## 2.2 模块划分

| 模块 | 文件 | 职责 |
|------|------|------|
| 路由控制 | `app.py` | HTTP 路由分发、请求/响应处理、任务状态管理 |
| AI 服务 | `ai_service.py` | DeepSeek API 调用、多线程异步处理、回调管理 |
| 文件存储 | `file_storage.py` | 报告文件的保存、列表、读取、路径安全 |
| 前端展示 | `templates/index.html` | 用户界面、交互逻辑、状态刷新 |

## 2.4 数据流图

```
用户输入 → [暂存] → 暂存区(内存字符串列表)
    ↓
[发送给 AI]
    ↓
┌─────────────────────────────────────┐
│ 1. 清空暂存区（释放 UI）             │
│ 2. 组装完整 Prompt                  │
│ 3. 启动后台线程                      │
│ 4. 返回 task_id 给前端               │
└─────────────────────────────────────┘
    ↓
后台线程:
┌─────────────────────────────────────┐
│ 1. 调用 DeepSeek API                │
│ 2. 成功 → 保存报告到 TXT 文件       │
│ 3. 失败 → 保存错误信息到 TXT 文件   │
│ 4. 更新任务状态为 completed          │
└─────────────────────────────────────┘
    ↓
前端轮询 → 状态完成 → 刷新报告列表 → 用户点击查看
```

## 2.5 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| Web 框架 | Flask | 轻量、灵活、适合小型项目 |
| AI API | DeepSeek (OpenAI 兼容) | 性价比高，兼容 OpenAI SDK |
| 异步方案 | threading | Python 内置，无需额外依赖 |
| 前端 | 原生 HTML/CSS/JS | 无重型框架依赖，更可控 |
| 数据存储 | TXT 文件 | 简单可靠，便于查看和备份 |
