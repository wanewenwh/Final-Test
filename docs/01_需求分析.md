# 01 需求分析

## 1.1 项目背景

在科研工作中，灵感往往以碎片化的形式突然出现。传统笔记工具要求用户在灵感来临时一次性完整记录，但科学遐想往往需要多次沉淀才能形成完整的问题描述。同时，将这些灵感转化为结构化研究方案需要大量的背景知识和分析时间。

本系统旨在解决这一问题，提供一个「碎片化记录 → 暂存累积 → AI 深度分析」的完整工具链。

## 1.2 用户角色

| 角色 | 描述 |
|------|------|
| 科研人员 | 需要快速记录科学灵感，并希望获得AI辅助的分析报告 |
| 学生 | 在学习过程中产生疑问，需要系统化的研究指导 |
| 科普爱好者 | 对科学问题有好奇心，希望深入探索自己的想法 |

## 1.3 功能需求

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F-01 | 灵感录入 | 用户可在文本框中输入多行科学遐想文本 | 高 |
| F-02 | 暂存累积 | 用户可多次暂存文本，累积在暂存区 | 高 |
| F-03 | 快捷操作 | 支持 Ctrl+Enter 快捷键暂存 | 中 |
| F-04 | 清空暂存 | 一键清空暂存区内容 | 中 |
| F-05 | AI 生成报告 | 将暂存区内容发送给 DeepSeek API 生成结构化报告 | 高 |
| F-06 | 异步不阻塞 | AI 生成过程中用户可继续输入新内容 | 高 |
| F-07 | 报告列表 | 按时间倒序展示所有已生成的报告 | 高 |
| F-08 | 报告查看 | 点击报告条目弹窗查看完整内容 | 高 |
| F-09 | 报告刷新 | 手动或自动刷新报告列表 | 中 |
| F-10 | 错误处理 | API 调用失败时保存错误信息到文件 | 高 |

## 1.4 非功能需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NF-01 | 响应时间 | 前端点击「发送」后应立即返回，不允许阻塞超过 1 秒 |
| NF-02 | 并发支持 | 支持连续发送多个请求，后台线程互不干扰 |
| NF-03 | 安全性 | 防止路径穿越攻击，API Key 不硬编码 |
| NF-04 | 兼容性 | 支持 Chrome/Firefox/Edge 现代浏览器 |
| NF-05 | 可维护性 | 代码采用 OOP 设计，数据与逻辑分离 |
